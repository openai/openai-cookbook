Operaciones alta\_otrocobro, editar\_otrocobro y leer\_otrocobro

Todos los campos mencionados a continuación son enviados al servidor de API en las operaciones de alta\_otrocobro y editar\_otrocobro, con el formato de propiedades de un objeto JSON como se muestra en los ejemplos respectivos.

Todos los campos son requeridos y deberán contener nulo ('') en caso de no tener valor.

En la tabla que se muestra a continuación se listan los campos requeridos, su tipo, su valores permitidos y si deben obligatoriamente contener un valor que no sea nulo ('').

|     |     |
| --- | --- |
| **Nombre del campo** | **Requerimientos** |
| idEmpresa | Identificador de la empresa en todo el sistema. Numérico entero. Obligatorio |
| idCobro | Identificador del Cobro de la Empresa. Numérico entero. Obligatorio en la edición. |
| fechaContable | Formato 'dd-mm-aaaa' o 'dd-mm-aa'. Obligatorio |
| descripcion | Alfanumérico. No más largo de 150 caracteres. Obligatorio |
| asientos | Arreglo. Obligatorio |
| fondos | Arreglo. Obligatorio |

**Arreglo asientos**

|     |     |
| --- | --- |
| idPlanCuenta | Alfanumérico. Número de Cuenta del Plan de Cuentas. Obligatorio |
| importe | Numérico con decimales. Positivo. Obligatorio |
| comentario | Alfanumérico. No más largo de 150 caracteres. Obligatorio |
| ccosto1 | Nombre del Centro de costos 1. |
| ccosto2 | Nombre del Centro de costos 2. |

**Arreglo fondos**

|     |     |
| --- | --- |
| idMedioCobro | Ver tabal de Medios de Cobro en Apéndice. Obligatorio |
| idPlanCuenta | Alfanumérico. Número de Cuenta del Plan de Cuentas. Obligatorio si el idMedioCobro es "Cheque Recibido Comun" o "Cheque Recibido Diferido" |
| Banco | Alfanumérico. No más largo de 20 caracteres. |
| nroCheque | Alfanumérico. No más largo de 20 caracteres. |
| fechaValidez | Formato 'dd-mm-aaaa' o 'dd-mm-aa'. Obligatorio si nroCheque es distinto de vacío. |
| importe | Numérico con decimales. Positivo. Obligatorio |
