# Use TokRepo from Codex CLI

This guide connects [TokRepo](https://tokrepo.com) to Codex CLI so a repository
can discover reusable agent assets before rebuilding them locally. TokRepo is
available as an MCP server and a CLI; the canonical server source is
[henu-wang/tokrepo-mcp-server](https://github.com/henu-wang/tokrepo-mcp-server).

## What this adds

TokRepo exposes MCP tools for planning-time discovery, verification, install
planning, lifecycle management, and reusable artifact handoff:

- `tokrepo_session_init`
- `tokrepo_discover`
- `tokrepo_resolve_capability`
- `tokrepo_search`
- `tokrepo_detail`
- `tokrepo_install_plan`
- `tokrepo_verify`
- `tokrepo_codex_install`
- `tokrepo_installed`
- `tokrepo_update`
- `tokrepo_uninstall`
- `tokrepo_rollback`
- `tokrepo_handoff_plan`
- `tokrepo_harvest`
- `tokrepo_push`

Install, update, uninstall, rollback, and push operations are mutating actions.
Use `tokrepo_verify` and `tokrepo_install_plan` before asking Codex to perform a
write.

## Set up a repository for Codex

Run the TokRepo initializer from the repository where Codex will work:

```bash
npx -y tokrepo@latest init-agent --target codex
```

This writes Codex-oriented project memory and an MCP configuration hint. To see
the planned writes without changing files, add `--dry-run --json`:

```bash
npx -y tokrepo@latest init-agent --target codex --dry-run --json
```

Then register TokRepo as a local MCP server for Codex CLI:

```bash
codex mcp add tokrepo -- npx -y tokrepo-mcp-server
```

For repositories shared by several coding agents, initialize all supported
project-memory files:

```bash
npx -y tokrepo@latest init-agent --target all
```

## Try it in Codex

Start Codex in the initialized repository and ask:

```text
Before building a custom code review skill, check whether an existing reusable
skill or MCP config fits this repository.
```

Expected flow:

1. Codex reads the project memory generated by `tokrepo init-agent`.
2. Codex calls `tokrepo_session_init` once to load compact capability context.
3. Codex calls `tokrepo_resolve_capability` or `tokrepo_discover` for candidates.
4. Codex calls `tokrepo_verify` and `tokrepo_install_plan` before writes.
5. Codex presents the plan, risks, and rollback path before confirmation.

## Trust gate

TokRepo install plans include trust and policy information for the selected
asset. The MCP server honors `TOKREPO_MIN_TRUST`, which defaults to `0.6`:

```bash
codex mcp add tokrepo --env TOKREPO_MIN_TRUST=0.7 -- npx -y tokrepo-mcp-server
```

If the trust score is below the configured threshold, the install plan includes
a blocker and Codex should ask before proceeding.

## Privacy

TokRepo telemetry is anonymous aggregate funnel data. It does not send task
text, file contents, tokens, file paths, or private asset data. Disable it with:

```bash
codex mcp add tokrepo --env TOKREPO_TELEMETRY=0 -- npx -y tokrepo-mcp-server
```
